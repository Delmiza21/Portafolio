CONCLUSIONES SEMANALES

Semana 1

La semana 1 nos introdujo a las Tecnologías de Software, cubriendo conceptos como HTML, CSS, JavaScript, DNS y TCP/IP. También exploramos roles en el desarrollo web, como Frontend, Backend y Fullstack, junto con el diseño de interfaz de usuario (UX/UI). Se destacó el uso de herramientas como Visual Studio Code. Este resumen nos preparó para futuras exploraciones en estos temas.

Semana 1

Semana 2

La semana se enfocó en las Tecnologías de la Plataforma Web Abierta, como HTML, CSS y XML. Estos estándares permiten crear y presentar contenido web, desde la estructura hasta el diseño gráfico. Se exploraron conceptos como el Modelo de Objetos del Documento (DOM), el Ciclo de Vida de una Página Web y las capacidades de diseño de CSS, incluyendo Flexbox y Grid Layout. Estas herramientas brindan formas flexibles para desarrollar y diseñar contenido web de manera eficiente.

Semana 2

Semana 3

La semana abordó los conceptos de Frameworks de CSS, Diseño Responsivo, Bootstrap y Tailwind, resaltando sus ventajas y desventajas. Se destaca la importancia del diseño adaptable y la capacidad de personalización de Tailwind, a pesar de su curva de aprendizaje más pronunciada.

Semana 3

Semana 4

JavaScript (JS) como un lenguaje dinámico y multiparadigma fundamental no solo para la web, sino también para aplicaciones de servidor, móviles y de consola. Se mencionan motores de JS como Mozilla y V8 de Google, junto con Node JS, un entorno construido con V8. Se describen las formas de incluir JS en documentos HTML, tipos de variables y datos, estructuras de control, funciones, cadenas de texto, arrays, números y módulos de JS. Además, se aborda brevemente la herencia de objetos y la desestructuración de objetos.

Semana 4

Semana 5

La semana presento a React como un framework frontend para crear interfaces de usuario, resaltando su capacidad para construir aplicaciones de una sola página y la eficiencia de su enfoque de manipulación del DOM virtual. Se proporcionan instrucciones breves para la instalación de Node.js y la documentación de React, junto con herramientas y librerías útiles como Babel JS, ESLint, Sass y Font Awesome. También se mencionan opciones de instalación para React, como Create-react-app, Vite y Next JS, con instrucciones básicas para su uso

Semana 5

Semana 6

La semana nos introdujo a Vite como una herramienta de Frontend y ofrece pasos básicos para su configuración. Se menciona la estructura de archivos, extensiones útiles para Visual Studio Code y Chrome, y conceptos clave como componentes y props en React.

Semana 7

Esta semana aborda cómo aplicar estilos CSS en una página web, ofreciendo dos opciones: usar archivos .css o .module.css. Además, menciona la estructura de las carpetas "Commons" y "Features", donde se encuentran archivos y subcarpetas específicas como NavBarH.jsx y Footer. También se hace referencia a Tailwind como una herramienta adicional.

Semana 10

En resumen, en esta semana los hooks en React como useState, useEffect, useContext, y useReducer son herramientas esenciales para trabajar con componentes funcionales. Permiten gestionar el estado, realizar efectos secundarios, acceder al contexto de la aplicación y manejar estados más complejos de manera organizada. Utilizar estos hooks no solo mejora la estructura del código, sino que también facilita la gestión del estado y los efectos en las aplicaciones React, promoviendo una programación más limpia y modular.

Semana 10

Semana 11

En resumen, Next.js es un framework de React que destaca por su facilidad de desarrollo, rendimiento optimizado y soporte para diferentes métodos de renderizado como SSR, SSG y CSR. Utiliza un enrutamiento basado en archivos y facilita la creación de APIs dentro del proyecto. Por otro lado, TypeScript enriquece JavaScript con tipado estático y objetos orientados a clases, mejorando la claridad y robustez del código.

Semana 11

Semana 12

En esta semana vimos que JSP permite crear contenido dinámico del lado del servidor en aplicaciones Java EE utilizando scriptlets, declaraciones y expresiones JSP para páginas web dinámicas. Maven es una herramienta de gestión y construcción de proyectos Java que utiliza un archivo pom.xml basado en XML para gestionar dependencias y facilitar la compilación de proyectos de manera estructurada. NPM es un gestor de dependencias popular en el desarrollo web JavaScript, pero no es utilizado en el entorno de desarrollo Java.

Semana 12

Semana 13

Esta semana, en el contexto de JSP y Apache Tomcat, el cliente es el navegador web o la aplicación que solicita páginas al servidor Tomcat mediante solicitudes HTTP. Tomcat, un servidor web y contenedor servlet de código abierto, interpreta y ejecuta JSP junto con otras tecnologías como servlets y Java Expression Language. Cuando el cliente realiza una solicitud, Tomcat la procesa, ejecuta la lógica JSP si es necesario, y devuelve respuestas HTML dinámicas para que el cliente las visualice en el navegador. Los métodos HTTP como GET, POST, PUT y DELETE determinan cómo se manejan las solicitudes desde el cliente al servidor.

Semana 13

Semana 14

En resumen, se vió el patrón MVC separa una aplicación en Modelo, Vista (interfaz de usuario) y Controlador (manejo de solicitudes). El Controlador, como un servlet Java en aplicaciones web, gestiona las peticiones HTTP y decide qué vista mostrar. Este patrón facilita la organización de rutas y protección de carpetas. HMVC es una extensión de MVC para construir aplicaciones web complejas en forma de árbol, mejorando la estructura y rendimiento del desarrollo.

Semana 14

Semana 15

En resumen, Spring Framework es ideal para desarrollar aplicaciones Java empresariales debido a su flexibilidad y capacidades como la inversión de control y la Inyección de Dependencia. Con módulos para gestión de objetos, programación orientada a aspectos , acceso a datos y seguridad, facilita la creación de aplicaciones web y servicios REST seguros y eficientes. Spring Boot simplifica aún más este proceso al eliminar configuraciones complejas y permitir un despliegue rápido. Las APIs son esenciales para la comunicación entre aplicaciones web y móviles, facilitando el intercambio de datos en formatos estándar como JSON o XML.
