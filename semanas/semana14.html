DEFINICIONES


Modelo-Vista-Controlador: El patrón MVC es un patrón de diseño arquitectónico que separa una aplicación en tres componentes principales:

  • Modelo (Model): Representa los datos y la lógica de negocio de la aplicación. Idealmente, el modelo no debería depender de la vista o del controlador.
  • Vista (View): Es responsable de la presentación de los datos al usuario y de la interacción con éste. En una aplicación web, la vista puede ser una página JSP, HTML o cualquier otro tipo de interfaz de usuario.
  • Controlador (Controller): Actúa como intermediario entre el modelo y la vista, maneja las solicitudes del usuario y actualiza el modelo según sea necesario. En una aplicación web, el controlador puede ser un servlet Java que procesa las solicitudes HTTP y luego determina qué vista mostrar al usuario.

En resumen crea rutas, protege carpetas y realiza sistema de rutas

HMVC (Hierarchical Model-View-Controller)

HMVC es una extensión del patrón MVC que se utiliza para construir aplicaciones web complejas y estructuradas en forma de árbol.
