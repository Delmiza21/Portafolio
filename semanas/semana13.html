TOMCAT Y JSP

En esta semana 13 hablaremos acerca de TOMCAT y JSP

DEFINICIONES


Cliente en JSP:
En JSP, el término "cliente" se refiere al navegador web o aplicación que solicita una página web al servidor Tomcat. El cliente envía una solicitud HTTP al servidor Tomcat que contiene la URL de la página JSP solicitada. El servidor Tomcat procesa esta solicitud, ejecuta la lógica de la página JSP si es necesario (como consultar una base de datos), y luego genera una respuesta HTML dinámica que se envía de vuelta al cliente para ser renderizada en el navegador web.

Apache Tomcat: Es un servidor web y contenedor servlet de código abierto desarrollado por la Apache Software Foundation. Tomcat actúa como un servidor web que sirve páginas web estáticas y dinámicas y también funciona como un contenedor servlet que implementa las especificaciones de Java Servlet, JavaServer Pages (JSP), y Java Expression Language (EL).
Relación entre JSP y Tomcat:Cuando un cliente solicita una página JSP a través de Tomcat, Tomcat es responsable de procesar esa solicitud, compilar el código JSP en un servlet Java, ejecutar ese servlet para generar contenido dinámico (como consultas a bases de datos o lógica de negocio), y finalmente enviar la respuesta HTML resultante de vuelta al cliente. Tomcat actúa como el servidor que interpreta y ejecuta el código JSP, facilitando así la generación dinámica de contenido web basado en Java
Metodos HTTP:GET, POST, PUT y DELETE se utilizan para determinar la forma en que se procesa una solicitud HTTP enviada desde el cliente al servidor.

  • GET: Este método se utiliza para solicitar datos del servidor. En JSP, cuando se recibe una solicitud GET, los parámetros de la solicitud pueden ser accesibles a través del objeto
  • POST: Este método se utiliza para enviar datos al servidor, generalmente a través de un formulario HTML en una página JSP. En este caso, los datos enviados por el cliente pueden ser accesibles a través del objeto
  • PUT y DELETE: Estos métodos HTTP son menos comunes en formularios HTML estándar, pero pueden ser utilizados en aplicaciones web más avanzadas o cuando se trabaja con APIs RESTful.
