JAVA y Spring Boot

En esta semana 15 hablaremos acerca de JAVA y Spring Boot

DEFINICIONES


Spring Framework: El Spring Framework proporciona un modelo de programación y configuración completo para aplicaciones empresariales modernas basadas en Java, sobre cualquier tipo de plataforma de implementación. Integra los diferentes patrones de diseño en la programación.

Patron de diseño: El patrón de diseño de Inyección de Dependencia (DI) implica la inversión de control, en la cual las propiedades de un objeto se introducen a través de un constructor, un setter, un servicio, u otro método. Este enfoque permite un control más flexible sobre el comportamiento de la aplicación, adaptándolo de manera conveniente a nuestras necesidades.

Spring Framework consiste en varios módulos que ofrecen una amplia gama de servicios:
  1. Contenedor de inversión de control: Gestiona el ciclo de vida de los objetos Java mediante la inyección de dependencias.
  2. Programación orientada a aspectos:Permite implementar rutinas transversales en la lógica de la aplicación.
  3. Acceso a datos: Facilita el trabajo con bases de datos relacionales y NoSQL usando JDBC y herramientas de mapeo objeto-relacional.
  4. Gestión de transacciones: Unifica APIs para coordinar transacciones en aplicaciones Java.
  5. Modelo vista controlador: Framework basado en HTTP y servlets para el desarrollo y personalización de aplicaciones web y servicios REST.
  6. Framework de acceso remoto: Permite la importación/exportación de objetos Java utilizando RMI, CORBA, HTTP y servicios web (SOAP).
  7. Autenticación y Autorización: Procesos de seguridad configurables con soporte para diversos estándares y protocolos a través de Spring Security (anteriormente Acegi).
  8. Administración Remota: Configura y gestiona objetos Java local o remotamente usando JMX.

SPRING BOOT:

Spring Boot es una herramienta que permite crear proyecto con Spring Framework, eliminando la complejidad de las configuraciones, instalación de dependencias además integrando un servidor de pruebas para desplegar la aplicación en desarrollo.

SERVICIOS API

API es acrónimo de Interfaz de Programación de Aplicaciones (del inglés Application Programming Interface), es un conjunto de protocolos y definiciones que pueden permitir que una aplicación se comunique con la otra. El formato de intercambio de datos normalmente es JSON o XML.

¿Porqué es importante un API?


  • Recepción y transmisión de datos entre aplicaciones.
  • Suministrar el servicio a las aplicaciones en la web
  • Suministrar el servicio a las aplicaciones en dispositivos móviles
